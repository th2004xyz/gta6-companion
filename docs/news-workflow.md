# 新闻运营工作流

本文档说明如何使用自动化脚本完成每日新闻内容的拉取、编辑与发布。

## 概览

整个流程分两个阶段：

1. **自动阶段**（你不用管）：GitHub Actions 每天 08:00（北京时间）自动从英文游戏媒体 RSS 抓取 GTA6 相关报道，生成草稿到 `news-drafts` 分支。
2. **人工阶段**（你的工作）：拉取草稿 → 编辑中文正文 → 发布到 main 分支 → Vercel 自动部署。

## 信源

自动抓取脚本（`scripts/fetch-news.ts`）从以下 5 个英文游戏媒体 RSS 抓取：

| 媒体 | RSS URL |
|------|---------|
| Eurogamer | `https://www.eurogamer.net/feed` |
| VGC | `https://www.videogameschronicle.com/feed/` |
| Push Square | `https://www.pushsquare.com/feeds/news` |
| GameSpot | `https://www.gamespot.com/feeds/mashup/` |
| IGN | `https://feeds.feedburner.com/ign/games-all` |

过滤关键词：`gta 6`、`gta6`、`grand theft auto 6`、`grand theft auto vi`、`rockstar`。

**日期过滤**：只抓取最近 3 天（今天、昨天、前天）的新闻，避免旧新闻反复出现。运行 `pull-drafts.bat` 时会自动清理本地超过 3 天且未编辑的草稿。已编辑的草稿不会被清理。如需调整天数，编辑 [scripts/fetch-news.ts](../scripts/fetch-news.ts) 顶部的 `MAX_AGE_DAYS` 常量。

> Rockstar 官方没有 RSS，但其动态会被上述媒体转载，所以采用聚合方式覆盖。

## 你每天的操作

### 第 1 步：拉取最新代码（开始编辑前必做）

打开 VS Code 终端，运行：

```bash
git pull origin main
```

**为什么要做这步**：确保本地与远程同步，避免后续推送时冲突。如果你之前推送过未编辑的草稿或其他改动，必须先拉取。

### 第 2 步：双击 `pull-drafts.bat` 拉取草稿

项目根目录下有一个 `pull-drafts.bat` 文件，**双击运行**即可。

正常输出示例：

```
====================================
  GTA6 News - 拉取最新草稿
====================================

[1/3] 从 GitHub 拉取最新数据...
[2/3] 检查 news-drafts 分支上的草稿...
  [+] 新草稿: src/content/news/2026-06-28-xxx.md
  [+] 新草稿: src/content/news/2026-06-28-yyy.md
[3/3] 完成！
共拉取 2 个新草稿。
```

按回车关闭窗口。草稿已下载到 `src/content/news/` 文件夹。

**如果窗口立即关闭或没反应**：在 VS Code 终端直接运行：
```bash
.\pull-drafts.bat
```

### 第 3 步：在 VS Code 编辑草稿

1. VS Code 左侧文件树展开：`src` → `content` → `news`
2. 双击任意 `.md` 文件打开

草稿文件结构如下：

```markdown
---
title: GTA 6 PS5 Disc Hopes Seemingly Dashed by Rockstar Source
title_en: GTA 6 PS5 Disc Hopes Seemingly Dashed by Rockstar Source
status: speculated
last_updated: 2026-06-27
date: 2026-06-27
author: Push Square
summary: Flying disc. GTA 6 will launch without a Blu-ray disc...
summary_en: Flying disc. GTA 6 will launch without a Blu-ray disc...
sources:
  - label:
      zh: Push Square
      en: Push Square
    url: https://www.pushsquare.com/...
---

<!-- AUTO-FETCHED DRAFT
  - This file was generated by scripts/fetch-news.ts from Push Square RSS.
  - Editor TODO:
    1. Translate `title` and `summary` fields to Chinese (keep title_en/summary_en as-is)
    2. Replace the Chinese placeholder below with a real Chinese summary/body
    3. Update `status` if needed (confirmed / speculated / leaked)
    4. Remove this comment block before publishing
-->

## 中文正文（待编辑）

> 此为自动抓取的英文草稿，请补充中文摘要与正文。

---

## Overview

A new report has cast doubt on hopes that a physical version of...

## Source

- Original: [GTA 6 PS5 Disc Hopes...](https://www.pushsquare.com/...)
```

**你要做的 4 件事**：

| # | 改什么 | 怎么改 |
|---|--------|--------|
| 1 | 删除 `<!-- AUTO-FETCHED DRAFT ... -->` 整段注释 | 选中删除（这是脚本的"我编辑完了"信号） |
| 2 | `title:` 字段翻译成中文 | 例：`title: GTA6 PS5 实体光盘版希望破灭` |
| 3 | `summary:` 字段翻译成中文 | 例：`summary: GTA6 首发将不包含蓝光实体光盘...` |
| 4 | 替换"## 中文正文（待编辑）"下的占位符 | 写真实中文内容 |

**英文部分完全不动**：`title_en`、`summary_en`、`## Overview`、`## Source` 是 RSS 抓取的英文原文，给英文用户看的，保持原样。

### 第 4 步：保存

按 **Ctrl+S** 保存文件。

### 第 5 步：双击 `publish-news.bat` 发布

脚本会自动：
1. 扫描 `src/content/news/` 下所有 `.md` 文件
2. 判断哪些已删除 `AUTO-FETCHED DRAFT` 注释（= 已编辑）
3. 只把已编辑的文件 commit + push 到 main 分支
4. 保留未编辑的文件在本地工作区

正常输出示例：

```
====================================
  GTA6 News - 发布已编辑的草稿
====================================

已编辑（将发布）: 2 个文件
  [+] src/content/news/2026-06-28-xxx.md
  [+] src/content/news/2026-06-28-yyy.md

未编辑（不发布，保留在工作区）: 5 个文件
  [-] src/content/news/2026-06-28-zzz.md
  ...

即将发布以上 2 个文件到 GitHub main 分支，
Vercel 会自动部署到线上。确认发布？(y/N)
```

输入 `y` 回车确认。看到 `[OK] 已发布到 GitHub！` 即成功。

### 第 6 步：验证线上

1-2 分钟后访问 <https://gta6.sohou.xyz/zh/news>，看新闻是否上线。

## 中文正文怎么写

3 种方案，按效率从高到低：

### 方案 A：只写中文摘要（最快，5 分钟/篇）

不翻译英文正文，只写 2-3 句中文摘要：

```markdown
## 中文摘要

据 Push Square 报道，GTA6 首发将不推出实体光盘版，仅以"盒装兑换码"形式发售。
Rockstar 未公开解释原因，业内人士推测是为了防止发售前泄露。

完整英文报道见下方。
```

### 方案 B：AI 辅助翻译（推荐，10 分钟/篇）

1. 复制草稿里 `## Overview` 下的英文正文
2. 粘贴给 ChatGPT / Claude / 豆包 等 AI，加提示词：
   ```
   把下面的英文游戏新闻翻译成中文，保持游戏术语准确：
   GTA6 / Vice City / Rockstar 等专有名词保留英文。

   [粘贴英文]
   ```
3. 复制 AI 翻译的中文，粘贴到 `## 中文正文（待编辑）` 位置
4. 快速校对，改掉明显错误
5. 删除 AUTO-FETCHED DRAFT 注释，保存

### 方案 C：完整人工翻译（最慢，质量最高）

逐句翻译英文正文。仅用于重大新闻（发售日、定价等）。

## 常见问题

### Q：未编辑的草稿怎么办？

不管它就行——未编辑的草稿会一直留在本地 `src/content/news/` 文件夹，但**不会上线上**。明天再运行 `pull-drafts.bat` 也不会重复拉取已存在的草稿。

想清理：在 VS Code 文件树右键删除那个 .md 文件即可。

### Q：推送时报 "non-fast-forward" 错误

说明远程 main 比你本地新（可能 GitHub Actions 又跑了一次）。解决：

```bash
git pull origin main --rebase
git push origin main
```

如果出现 `CONFLICT`，运行：
```bash
git checkout --ours src/content/news/<冲突的文件.md>
git add src/content/news/<冲突的文件.md>
git rebase --continue
git push origin main
```

### Q：误报的新闻怎么处理

约 4-5% 的草稿只是文中提到 Rockstar，主题跟 GTA6 无关（如"Oldschool Runescape shows up in..."）。直接在 VS Code 文件树右键删除即可。

### Q：想立即测试 GitHub Actions

访问 <https://github.com/th2004xyz/gta6-companion/actions/workflows/fetch-news.yml>，点击 "Run workflow" 按钮手动触发。

### Q：想加更多 RSS 源

编辑 [scripts/fetch-news.ts](../scripts/fetch-news.ts) 顶部的 `SOURCES` 数组，提交到 main 即可。

### Q：news-drafts 分支可以删除吗

可以。明天 08:00 GitHub Actions 会自动从最新 main 重建。但**你的编辑必须落到 main**，不能停留在 news-drafts——否则重建时会丢失。
